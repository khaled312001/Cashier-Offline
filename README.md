# كاشير أوفلاين — Cashier Offline

نظام نقاط بيع (POS) ديسكتوب يعمل **أوفلاين بالكامل** لكل أنواع المحلات (سوبر ماركت، مطعم/كافيه، ريتيل، صيدلية، مكتبة)، بواجهة عربية RTL، مع نظام ترخيص (تجريبي/شهري/سنوي/دائم) يعمل بدون إنترنت.

التقنية: **Electron + React + Vite + TypeScript + TailwindCSS + SQLite (better-sqlite3) + Drizzle ORM**.

---

## المتطلبات
- Node.js 20+ و npm
- ويندوز (للأجهزة والطباعة) — التطوير يعمل على أي نظام
- لبناء الـ native module: أدوات بناء C++ (Visual Studio Build Tools) إن لم تتوفر النسخ الجاهزة

## التشغيل أول مرة (تطوير)
```bash
npm install            # يثبّت الحزم ويعيد بناء better-sqlite3 لإلكترون (postinstall)
npm run db:generate    # يولّد ملفات الـ migrations من المخطط
npm run dev            # يشغّل التطبيق (نافذة عربية RTL)
```
> قاعدة البيانات والـ migrations والـ seed تُنشأ تلقائيًا عند أول إقلاع داخل `.dev-userdata/`.

**الدخول الافتراضي:** المستخدم `admin` — الرقم السري `1234`
**الترخيص:** اضغط "بدء النسخة التجريبية" (30 يوم) للبدء فورًا.

## البناء وإنشاء ملف التثبيت (ويندوز)
```bash
npm run db:generate    # تأكد من وجود الـ migrations
npm run dist           # يبني التطبيق وينشئ ملف تثبيت NSIS في مجلد release/
```
الناتج: `release/Cashier Offline-Setup-1.0.0-x64.exe` — يُثبَّت بدون صلاحيات مدير، ويخزّن البيانات في `%APPDATA%\Cashier Offline`.

## نظام الترخيص (للشركة البائعة)
1. ولّد زوج المفاتيح مرة واحدة (يُنشئ المفتاح العام داخل التطبيق والخاص يبقى لديك):
   ```bash
   npm run license:keygen
   ```
2. اطلب من العميل **معرّف الجهاز (Machine ID)** من شاشة التفعيل، ثم ولّد له ترخيصًا:
   ```bash
   npm run license:gen -- --customer CUST-001 --name "سوبر ماركت السلام" --type monthly --machine-id <ID>
   ```
   الأنواع: `trial | monthly | annual | perpetual`. للمدة المخصّصة: `--duration 30d|6m|1y`.
3. ابعث للعميل **مفتاح التفعيل** (base64) المطبوع، أو ملف `.lic`، ليُلصق في شاشة التفعيل.

> ⚠️ مجلد `tools/license-cli/keys/` يحتوي المفتاح الخاص — لا تشحنه ولا ترفعه إطلاقًا (مستبعد في .gitignore).

## البنية
- `db/schema/` نموذج البيانات (Drizzle) · `db/fts.sql` بحث عربي FTS5
- `src/main/` عملية Electron الرئيسية: DB، IPC، الخدمات، الأجهزة، النسخ، الترخيص
- `src/preload/` جسر آمن مكتوب النوع → `window.api`
- `src/renderer/` واجهة React (شاشات البيع، الأصناف، المخزون، العملاء، التقارير، الوردية، الإعدادات، النسخ)
- `src/shared/` أنواع وثوابت ومنطق مشترك (النقود بالقروش، الصلاحيات، عقد IPC)
- `tools/license-cli/` أداة توليد التراخيص (داخلية)

## ملاحظات
- **النقود** تُخزَّن دائمًا كأعداد صحيحة بالقروش (1 جنيه = 100).
- الطباعة في التطوير تُكتب كملف نصّي في `logs/receipts/` (Dev Sink) بدلًا من طابعة حقيقية.
- الفاتورة الإلكترونية (ETA) خارج النطاق حاليًا، لكن النموذج جاهز لإضافتها لاحقًا.

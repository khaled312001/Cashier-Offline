# دليل إدارة التراخيص والاشتراكات — لشركة برمجلي

كيف تبيع برمجلي البرنامج، تُصدر أكواد التفعيل، وتتحكم في الاشتراكات (شهري/سنوي/دائم) — كل ذلك **أوفلاين بالكامل** بدون سيرفر.

---

## 1) الفكرة باختصار

النظام يعتمد على **توقيع رقمي (Ed25519)**:
- عندك **مفتاح خاص (Private Key)** سري واحد فقط، يبقى على جهاز برمجلي ولا يُشحن أبدًا.
- داخل البرنامج عند كل عميل **مفتاح عام (Public Key)** مضمَّن، يتحقق فقط من صحة الترخيص ولا يستطيع إنشاء تراخيص.
- كل ترخيص = ملف/كود موقّع يحتوي: اسم العميل، النوع (تجريبي/شهري/سنوي/دائم)، تاريخ الانتهاء، **ومعرّف الجهاز (Machine ID)**.
- البرنامج يتحقق محليًا: التوقيع صحيح؟ + الجهاز مطابق؟ + لم تنتهِ الصلاحية؟ + الساعة لم تُرجَع للخلف؟ → يشتغل.

**لا يحتاج إنترنت إطلاقًا** — التحقق رياضي بالكامل على جهاز العميل.

---

## 2) خطوة لمرة واحدة: توليد مفاتيح برمجلي

على جهاز برمجلي فقط:
```bash
npm run license:keygen
```
ينتج:
- `tools/license-cli/keys/license_private.pem` ← **سري جدًا**. احتفظ به في مكان آمن (خزنة/مدير كلمات مرور). لو ضاع، تفقد القدرة على إصدار تراخيص جديدة. لو تسرّب، أي شخص يقدر يزوّر تراخيص.
- `resources/keys/license_public.pem` ← يُدمج تلقائيًا داخل البرنامج عند البناء (آمن للنشر).

> ⚠️ بمجرد توليد المفاتيح وبناء نسخة عند العملاء، **لا تغيّر المفتاح** — وإلا كل التراخيص القديمة تبطل.

---

## 3) دورة بيع عميل جديد (خطوة بخطوة)

1. العميل يثبّت البرنامج.
2. أول تشغيل: يظهر له خيار **"بدء النسخة التجريبية (30 يوم)"** ليجرّب فورًا.
3. عند الشراء: العميل يفتح شاشة التفعيل وينسخ لك **معرّف الجهاز (Machine ID)** (زر "نسخ").
4. برمجلي تولّد له كود الترخيص حسب الباقة:

```bash
# اشتراك شهري
npm run license:gen -- --customer C-1001 --name "سوبر ماركت السلام" --type monthly --machine-id <MACHINE_ID>

# اشتراك سنوي
npm run license:gen -- --customer C-1001 --name "سوبر ماركت السلام" --type annual --machine-id <MACHINE_ID>

# شراء دائم (لا ينتهي)
npm run license:gen -- --customer C-1001 --name "سوبر ماركت السلام" --type perpetual --machine-id <MACHINE_ID>

# مدة مخصصة (مثلاً 3 شهور)
npm run license:gen -- --customer C-1001 --name "..." --type monthly --duration 3m --machine-id <MACHINE_ID>
```

5. الأمر يطبع **كود تفعيل (نص base64 طويل)** وملف `.lic`. ابعت أيًّا منهما للعميل (واتساب/إيميل).
6. العميل يلصق الكود في شاشة التفعيل → "تفعيل" → يشتغل فورًا. النوع والأيام المتبقية تظهر له.

---

## 4) كيف نتحكم في الاشتراكات (المهم)

التحكم يكون عبر **تاريخ انتهاء مضمَّن في الكود الموقّع** — مش عبر سيرفر يوقف البرنامج عن بُعد.

| الباقة | السلوك |
|---|---|
| **تجريبي** | 30 يوم من أول تشغيل، ثم يتوقف البيع حتى التفعيل. |
| **شهري** | يشتغل حتى تاريخ الانتهاء، ثم **فترة سماح 7 أيام** (يشتغل مع تنبيه تجديد)، ثم يتوقف البيع. |
| **سنوي** | نفس الشهري لكن سنة كاملة. |
| **دائم** | لا ينتهي أبدًا (مربوط بالجهاز فقط). |

**التجديد** = تولّد كودًا جديدًا بنفس `--machine-id` وتاريخ انتهاء جديد، والعميل يلصقه. ثوانٍ معدودة.

البرنامج ينبّه العميل تلقائيًا قبل الانتهاء (عند ≤14 يوم وعند ≤3 أيام).

### الحماية المدمجة
- **ربط بالجهاز**: الكود يعمل على الجهاز اللي طلبه فقط. نسخه لجهاز تاني → "ترخيص غير صالح".
- **منع تزوير**: أي تعديل في النوع أو التاريخ يُبطل التوقيع فورًا.
- **منع تلاعب الساعة**: إرجاع ساعة الويندوز للخلف يُكتشف ويُرفض.

---

## 5) سجل العملاء (للمتابعة الإدارية)

النظام أوفلاين فلا يحفظ سجلًا مركزيًا. احتفظ بجدول بسيط (Excel/Sheet):

| كود العميل | الاسم | الهاتف | Machine ID | الباقة | تاريخ التفعيل | تاريخ الانتهاء |
|---|---|---|---|---|---|---|
| C-1001 | سوبر ماركت السلام | 010xxxx | a1b2... | شهري | 2026-06-01 | 2026-07-01 |

> نصيحة: ضع تذكيرًا قبل الانتهاء بـ3 أيام لكل عميل. ملفات `.lic` تُحفظ داخل `tools/license-cli/` — نظّمها بمجلد لكل عميل.

---

## 6) أسئلة شائعة

**العميل غيّر الجهاز / فورمت؟** يبعت المعرّف الجديد وتولّد كودًا جديدًا (سياسة برمجلي: مجاني خلال نفس الاشتراك).

**العميل عنده جهازين؟** كل جهاز يحتاج كودًا خاصًا (احسبها في التسعير: ترخيص لكل جهاز).

**انتهى الاشتراك ولم يجدد؟** فترة السماح 7 أيام تغطي التأخير. بعدها يتوقف البيع فقط (بياناته سليمة)، ويرجع فورًا بلصق كود التجديد.

**أوقف عميل لم يدفع عن بُعد؟** لا (أوفلاين بلا سيرفر) — التحكم عبر عدم التجديد.

---

## 7) ملخص الأوامر

| الغرض | الأمر |
|---|---|
| توليد المفاتيح (مرة واحدة) | `npm run license:keygen` |
| ترخيص شهري | `npm run license:gen -- --customer <ID> --name "<الاسم>" --type monthly --machine-id <MID>` |
| سنوي / دائم | `... --type annual` أو `... --type perpetual` |
| مدة مخصصة | `... --type monthly --duration 90d` |
| تجديد | نفس الأمر بنفس `--machine-id` |

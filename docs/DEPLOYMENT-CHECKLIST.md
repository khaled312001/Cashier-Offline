# قائمة التحقق قبل الإصدار — Deployment Checklist

قائمة يتبعها فريق **برمجلي** قبل كل إصدار/تسليم لضمان نسخة كاملة وخالية من الأخطاء.

---

## 1. الكود والجودة

- [ ] `npm run typecheck` ينجح بدون أخطاء (node + web).
- [ ] `npm run build` ينجح بدون أخطاء.
- [ ] الاختبار الشامل ينجح بالكامل (الهدف: **0 فشل**):
      ```
      npm run build
      # على هذا الجهاز: ألغِ ELECTRON_RUN_AS_NODE
      npx electron . --smoke      (أو: env -u ELECTRON_RUN_AS_NODE npx electron . --smoke)
      ```
      راجع `smoke-results.txt` — يجب أن يكون `fail=0`.
- [ ] لا توجد مفاتيح/أسرار في الكود؛ المفتاح الخاص للتراخيص **غير مُضمَّن** (راجع `.gitignore`).

## 2. مفتاح الترخيص (حرج — لا تتجاوزه)

- [ ] `npm run verify:keys` ينجح (المفتاح العام صالح ومطابق للخاص).
- [ ] **لم يُعد توليد زوج المفاتيح** بعد أول إصدار (يُبطل كل تراخيص العملاء).
- [ ] ملف `resources/keys/license_public.fingerprint` مُسجَّل وثابت بين الإصدارات.
- [ ] **لوحة التراخيص توقّع بالمفتاح الكنسي نفسه** المطابق للمفتاح العام داخل التطبيق. اللوحة الآن تزرع المفتاح تلقائيًا من `tools/license-manager/resources/keys/vendor_private.pem` (مُجمّع داخلها) عند أول تشغيل. إن ظهر للعميل خطأ «الترخيص غير صالح لهذا الجهاز» رغم تطابق معرّف الجهاز ⇐ اللوحة وقّعت بمفتاح مختلف: استبدل `%APPDATA%/barmagly-license-manager/keys/private.pem`+`public.pem` بالمفتاح الكنسي وأعد إصدار الترخيص.

## 3. الإعدادات الافتراضية

- [ ] نوع النشاط الافتراضي مناسب (أو يُضبط عند أول تشغيل عند العميل).
- [ ] العملة EGP والضريبة الافتراضية صحيحة.
- [ ] حساب المدير الافتراضي (admin / 1234) — يُنبَّه العميل لتغييره فورًا.
- [ ] النسخ الاحتياطي التلقائي مُفعّل (كل ٦ ساعات).

## 4. الطباعة والأجهزة

- [ ] تظهر الطابعات في **الإعدادات** ويمكن اختيار طابعة الإيصال/الملصق.
- [ ] طباعة تجريبية للإيصال تظهر العربية و QR بشكل صحيح.
- [ ] (للمطاعم) تذكرة المطبخ (KOT) تُطبع لأقسام المطبخ.

## 5. البناء والتغليف (Installer)

- [ ] `npm run dist` ينتج `release/Cashier Offline-Setup-x.x.x-x64.exe`.
- [ ] رقم الإصدار في `package.json` مُحدَّث.
- [ ] (إن وُجدت لوحة التراخيص) بناء installer اللوحة كذلك.
- [ ] تشغيل الـ installer على جهاز ويندوز نظيف:
  - [ ] التثبيت يكتمل (مع تجاوز SmartScreen: More info → Run anyway).
  - [ ] أول تشغيل: التفعيل/التجريبي يعمل، تسجيل الدخول يعمل.
  - [ ] دورة بيع كاملة: فتح وردية ← بيع ← دفع ← طباعة ← إغلاق وردية.

## 6. اختبار الترقية

- [ ] تثبيت النسخة الجديدة **فوق** نسخة قديمة على نفس الجهاز.
- [ ] التأكد من بقاء البيانات (المنتجات، الفواتير، العملاء) بعد الترقية.
- [ ] عدم الحاجة لإعادة التفعيل بعد الترقية.

## 7. حزمة التسليم للعميل

- [ ] ملف الـ installer.
- [ ] `INSTALLATION-GUIDE-AR.md` (التثبيت + تجاوز SmartScreen).
- [ ] `ACTIVATION-GUIDE-AR.md` (التفعيل والتجديد).
- [ ] `OPERATION-MANUAL-AR.md` (دليل التشغيل اليومي).
- [ ] بيانات الدعم الفني لبرمجلي.

---

## ملاحظات تشغيلية

- البناء يستخدم **CommonJS** للـ main/preload (لا تُضِف `"type": "module"` في `package.json` — يكسر تحميل الـ main تحت Electron).
- `better-sqlite3` أداة أصلية: عند تغيير إصدار Electron شغّل `npm run rebuild`.
- ETA (الفاتورة الإلكترونية) وتعدد الفروع مُهيّآن في قاعدة البيانات لكنهما خارج نطاق هذا الإصدار.
